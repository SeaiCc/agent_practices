# 使用 Claude Agent SDK 构建智能体

**发布时间：** 2025 年 9 月 29 日  
**来源：** https://claude.com/blog/building-agents-with-the-claude-agent-sdk

---

Claude Agent SDK 是一组工具，帮助开发者在 Claude Code 之上构建强大的智能体。在本文中，我们将介绍如何入门并分享最佳实践。

去年，我们与客户分享了 [构建高效智能体](https://www.anthropic.com/engineering/building-effective-agents) 的经验。此后，我们发布了 [Claude Code](https://claude.com/product/claude-code)，这是一个智能体编码解决方案，最初是为了支持 Anthropic 的开发者生产力而构建的。

在过去的几个月中，Claude Code 已远不止是一个编码工具。在 Anthropic，我们一直 [用它](https://www.anthropic.com/news/how-anthropic-teams-use-claude-code) 进行深度研究、视频创作和笔记记录，以及无数其他非编码应用。事实上，它已经开始驱动我们几乎所有的重大智能体循环。

换句话说，驱动 Claude Code 的智能体 harness（Claude Code SDK）也可以驱动许多其他类型的智能体。为了反映这一更广泛的愿景，我们将 Claude Code SDK 更名为 **Claude Agent SDK**。

在本文中，我们将介绍为什么构建 Claude Agent SDK、如何使用它构建自己的智能体，并分享我们团队自己的部署中出现的最佳实践。

## 给 Claude 一台计算机

Claude Code 背后的 [关键设计原则](https://www.youtube.com/watch?v=vLIDHi-1PVU) 是 Claude 需要程序员每天使用的相同工具。它需要能够查找代码库中的适当文件、编写和编辑文件、进行代码检查、运行它、调试、编辑，有时需要反复采取这些行动直到代码成功。

我们发现，通过让 Claude 访问用户的计算机（通过终端），它拥有了像程序员一样编写代码所需的一切。

但这也使得 Claude Code 中的 Claude 在*非*编码任务中非常有效。通过给它运行 bash 命令、编辑文件、创建文件和搜索文件的工具，Claude 可以读取 CSV 文件、搜索网络、构建可视化、解释指标，以及做各种其他数字工作——简而言之，用计算机创建通用智能体。Claude Agent SDK 背后的关键设计原则是给你的智能体一台计算机，让它们像人类一样工作。

## 创建新型智能体

我们相信让 Claude 拥有计算机能够解锁构建比以前更有效的智能体的能力。例如，使用我们的 SDK，开发者可以构建：

-   **金融智能体**：构建能够理解你的投资组合和目标，并通过访问外部 API、存储数据和运行代码进行计算来帮助你评估投资的智能体。
-   **个人助理智能体**：构建能够帮助你预订旅行和管理日历，以及预约、整理简报等的智能体，通过连接到你的内部数据源并跨应用程序跟踪上下文。
-   **客户服务智能体**：构建能够处理高度模糊的用户请求（如客户服务工单）的智能体，通过收集和审查用户数据、连接外部 API、与用户来回通信并在需要时升级到人工。
-   **深度研究智能体**：构建能够在大型文档集合中进行全面研究的智能体，通过搜索文件系统、分析和综合来自多个来源的信息、跨文件交叉引用数据，并生成详细报告。

还有很多。在核心层面，SDK 为你提供构建智能体的基本单元，用于任何你想要自动化的工作流。

## 构建你的智能体循环

在 Claude Code 中，Claude 通常在特定的反馈循环中运行：收集上下文 -> 采取行动 -> 验证工作 -> 重复。

智能体通常在特定的反馈循环中运行：收集上下文 -> 采取行动 -> 验证工作 -> 重复。

这提供了一种思考其他智能体及其应被赋予的能力的有用方式。为了说明这一点，我们将以如何在 Claude Agent SDK 中构建电子邮件智能体为例进行讲解。

## 收集上下文

开发智能体时，你希望给它不仅仅是提示：它需要能够获取和更新自己的上下文。以下是 SDK 中的功能如何帮助。

### **智能体搜索和文件系统**

文件系统代表*可以*被拉入模型上下文的信息。

当 Claude 遇到大文件（如日志或用户上传的文件）时，它将通过使用像 `grep`和`tail` 这样的 bash 脚本来决定如何将这些文件加载到上下文中。本质上，智能体的文件夹和文件结构成为一种形式的 [上下文工程](http://anthropic.com/news/context-management)。

我们的电子邮件智能体可能将之前的对话存储在名为"Conversations"的文件夹中。这将允许它在被问及这些对话时搜索之前的内容以获取上下文。

### **语义搜索**

[语义搜索](https://www.anthropic.com/news/contextual-retrieval) 通常比智能体搜索更快，但不太准确、更难维护、也不太透明。它涉及"分块"相关上下文，将这些块嵌入为向量，然后通过查询这些向量来搜索概念。鉴于其局限性，我们建议从智能体搜索开始，仅在需要更快结果或更多变体时才添加语义搜索。

### **子智能体**

Claude Agent SDK 默认支持子智能体。[子智能体](https://docs.claude.com/en/api/agent-sdk/subagents) 主要有两个用途。首先，它们实现并行化：你可以启动多个子智能体同时处理不同任务。其次，它们帮助管理上下文：子智能体使用自己隔离的上下文窗口，只将相关信息发送回编排者，而不是完整上下文。这使得它们非常适合需要筛选大量信息（其中大部分可能无用）的任务。

在设计我们的电子邮件智能体时，我们可能会给它一个"搜索子智能体"功能。电子邮件智能体然后可以并行启动多个搜索子智能体——每个对你的电子邮件历史记录运行不同的查询——并让它们只返回相关摘录而不是完整的电子邮件线程。

### **压缩**

当智能体长时间运行时，上下文维护变得至关重要。Claude Agent SDK 的压缩功能在上下文限制接近时自动总结之前的消息，因此你的智能体不会耗尽上下文。这建立在 Claude Code 的 [压缩斜杠命令](https://docs.claude.com/en/docs/claude-code/sdk/sdk-slash-commands#%2Fcompact-compact-conversation-history) 之上。

## 采取行动

一旦收集了上下文，你希望给智能体灵活的方式采取行动。

### **工具**

[工具](https://www.anthropic.com/engineering/writing-tools-for-agents) 是你智能体执行的主要构建块。工具在 Claude 的上下文窗口中很突出，使它们成为 Claude 在决定如何完成任务时将考虑的主要行动。这意味着你应该有意识地设计工具以最大化上下文效率。你可以在我们的博客文章 [为智能体编写高效工具——与智能体一起](https://www.anthropic.com/engineering/writing-tools-for-agents) 中查看更多最佳实践。

因此，你的工具应该是你希望智能体采取的主要行动。了解如何在 Claude Agent SDK 中制作 [自定义工具](https://docs.claude.com/en/api/agent-sdk/custom-tools)。

对于我们的电子邮件智能体，我们可能会定义像"`fetchInbox`"或"`searchEmails`"这样的工具作为智能体的主要、最频繁的行动。

### **Bash & 脚本**

Bash 作为通用工具有用，允许智能体使用计算机做灵活的工作。

在我们的电子邮件智能体中，用户可能在附件中存储了重要信息。Claude 可以编写代码来下载 PDF、将其转换为文本，并通过调用以下命令在其中搜索有用信息，如下图所示：

```shell
pdftotext document.pdf - | grep -n "invoice" | tail -10
```

### **代码生成**

Claude Agent SDK 擅长代码生成——这是有原因的。代码精确、可组合、无限可重用，使其成为需要可靠执行复杂操作的智能体的理想输出。

构建智能体时，考虑：哪些任务将受益于作为代码表达？通常，答案会解锁显著能力。

例如，我们最近在 [Claude.AI](http://claude.ai/redirect/website.v1.bdb29daa-1a07-41ec-87f6-579dc33634bd) 中推出的 [文件创建](https://www.anthropic.com/news/create-files) 功能完全依赖代码生成。Claude 编写 Python 脚本来创建电子表格、PowerPoint 演示文稿和 Word 文档，确保一致格式和复杂功能，这是通过其他方式难以实现的。

### **MCPs**

[模型上下文协议](https://modelcontextprotocol.io/)（MCP）提供到外部服务的标准化集成，自动处理身份验证和 API 调用。这意味着你可以将智能体连接到 Slack、GitHub、Google Drive 或 Asana 等工具，而无需编写自定义集成代码或自己管理 OAuth 流程。

对于我们的电子邮件智能体，我们可能想要 `search Slack messages` 来理解团队上下文，或 `check Asana tasks` 来查看是否已有人被分配处理客户请求。使用 MCP 服务器，这些集成开箱即用——你的智能体可以简单地调用像 search_slack_messages 或 get_asana_tasks 这样的工具，MCP 处理其余部分。

不断增长的 [MCP 生态系统](https://github.com/modelcontextprotocol/servers) 意味着你可以将新功能快速添加到智能体，因为预构建的集成变得可用，让你专注于智能体行为。

## 验证你的工作

Claude Code SDK 通过评估其工作来完成智能体循环。能够检查和改进自己输出的智能体从根本上更可靠——它们在错误累积之前捕捉错误，在偏离时自我纠正，并随着迭代变得更好。

关键是给 Claude 评估其工作的具体方式。以下是我们发现有效的三种方法：

### **定义规则**

最好的反馈形式是提供明确定义的输出规则，然后解释哪些规则失败以及为什么。

[代码检查](https://stackoverflow.com/questions/8503559/what-is-linting) 是规则反馈的优秀形式。反馈越深入越好。例如，通常生成 TypeScript 并检查它比生成纯 JavaScript 更好，因为它为你提供多个额外的反馈层。

生成电子邮件时，你可能希望 Claude 检查电子邮件地址是否有效（如果不是，抛出错误）以及用户是否之前给他们发送过电子邮件（如果是，抛出警告）。

### **视觉反馈**

当使用智能体完成视觉任务（如 UI 生成或测试）时，视觉反馈（以截图或渲染形式）可能很有帮助。例如，如果发送带有 HTML 格式的电子邮件，你可以截屏生成的电子邮件并将其提供给模型进行视觉验证和迭代细化。然后模型可以检查视觉输出是否符合要求。

例如：

-   **布局** - 元素是否正确定位？间距是否适当？
-   **样式** - 颜色、字体和格式是否按预期显示？
-   **内容层次** - 信息是否按正确顺序呈现并适当强调？
-   **响应性** - 它是否看起来破损或拥挤？（尽管单个截图的视口信息有限）

使用像 Playwright 这样的 MCP 服务器，你可以自动化这个视觉反馈循环——截取渲染 HTML 的截图、捕获不同视口大小，甚至测试交互元素——都在你的智能体工作流内。

### **LLM 作为评判者**

你还可以让另一个语言模型根据模糊规则"评判"智能体的输出。这通常不是非常健壮的方法，并且可能有重大的延迟权衡，但对于任何性能提升都值得成本的应用，它可能很有帮助。

我们的电子邮件智能体可能有单独的子智能体评判其草稿的语气，看看它们是否与用户之前的消息相适应。

## 测试和改进你的智能体

在运行几次智能体循环后，我们建议测试你的智能体，并确保它为其任务做好充分准备。改进智能体的最好方法是仔细查看其输出，特别是它失败的情况，并设身处地：它是否有 [正确的工具](https://www.anthropic.com/engineering/writing-tools-for-agents) 来完成工作？

以下是评估智能体是否做好充分准备时的一些其他问题：

-   如果你的智能体误解了任务，它可能缺少关键信息。你能改变搜索 API 的结构使其更容易找到它需要知道的吗？
-   如果你的智能体反复失败于任务，你能在工具调用中添加正式规则来识别和修复失败吗？
-   如果你的智能体无法修复错误，你能给它更有用或更有创意的工具以不同方式接近问题吗？
-   如果你的智能体性能随着添加功能而变化，基于客户使用构建代表性测试集用于程序化评估（或 evals）。

## 入门

Claude Agent SDK 通过让 Claude 访问计算机（在那里它可以编写文件、运行命令和迭代工作）使构建自主智能体更容易。

记住智能体循环（收集上下文、采取行动和验证工作），你可以构建可靠、易于部署和迭代的智能体。

你今天就可以 [开始使用](https://docs.claude.com/en/api/agent-sdk/overview) Claude Agent SDK。对于已经在 SDK 上构建的开发者，我们建议按照 [此指南](https://docs.claude.com/en/docs/claude-code/sdk/migration-guide) 迁移到最新版本。

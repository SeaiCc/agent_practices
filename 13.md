# 超越权限提示：让 Claude Code 更安全、更自主

发表于 2025 年 10 月 20 日

Claude Code 的新沙盒功能——bash 工具和 Web 版 Claude Code——通过启用两个边界（文件系统和网络隔离）来减少权限提示并提高用户安全性。

---

在 [Claude Code](https://www.claude.com/product/claude-code) 中，Claude 与你一起编写、测试和调试代码，导航你的代码库，编辑多个文件，并运行命令来验证其工作。给予 Claude 如此多的代码库和文件访问权限可能会引入风险，特别是在提示注入的情况下。

为帮助解决这个问题，我们在 Claude Code 中引入了两个基于沙盒的新功能，它们旨在为开发者提供更安全的工作环境，同时让 Claude 更自主地运行，减少权限提示。在我们内部使用中，我们发现沙盒安全地将权限提示减少了 84%。通过定义 Claude 可以自由工作的固定边界，它们提高了安全性和自主性。

### 在 Claude Code 上保持用户安全

Claude Code 基于权限模型运行：默认情况下是只读的，这意味着在进行修改或运行任何命令之前会请求权限。有一些例外：我们自动允许安全的命令如 echo 或 cat，但大多数操作仍需要明确批准。

不断点击"批准"会减慢开发周期，并可能导致"批准疲劳"，用户可能不会密切关注他们批准的内容，反过来使开发更不安全。

为了解决这个问题，我们为 Claude Code 推出了沙盒功能。

## 沙盒：更安全、更自主的方法

沙盒创建预定义的边界，Claude 在其中可以更自由地工作，而不是为每个操作请求权限。启用沙盒后，你会获得大大减少的权限提示和更高的安全性。

我们的沙盒方法建立在操作系统级功能之上，以启用两个边界：

1. **文件系统隔离**，确保 Claude 只能访问或修改特定目录。这对于防止被提示注入的 Claude 修改敏感系统文件尤为重要。
2. **网络隔离**，确保 Claude 只能连接到批准的服务器。这防止被提示注入的 Claude 泄露敏感信息或下载恶意软件。

值得注意的是，有效的沙盒需要*同时*具备文件系统和网络隔离。没有网络隔离，受损的智能体可能会泄露敏感文件如 SSH 密钥；没有文件系统隔离，受损的智能体可以轻松逃脱沙盒并获得网络访问权限。通过同时使用这两种技术，我们才能为 Claude Code 用户提供更安全、更快的智能体体验。

### Claude Code 中的两个新沙盒功能

#### 沙盒 bash 工具：无需权限提示的安全 bash 执行

我们推出 [一个新的沙盒运行时](https://docs.claude.com/en/docs/claude-code/sandboxing)，作为研究预览版 beta 可用，让你精确定义智能体可以访问哪些目录和网络主机，无需启动和管理容器的开销。这可以用于沙盒任意进程、智能体和 MCP 服务器。它也作为 [开源研究预览](https://github.com/anthropic-experimental/sandbox-runtime) 可用。

在 Claude Code 中，我们使用此运行时来沙盒 bash 工具，允许 Claude 在你设置的定义限制内运行命令。在安全沙盒内，Claude 可以更自主地运行，安全地执行命令而无需权限提示。如果 Claude 尝试访问沙盒*外部*的内容，你会立即收到通知，并可以选择是否允许它。

我们在此基础上构建了操作系统级原语，如 [Linux bubblewrap](https://github.com/containers/bubblewrap) 和 MacOS seatbelt，在操作系统级强制执行这些限制。它们不仅覆盖 Claude Code 的直接交互，还覆盖命令生成的任何脚本、程序或子进程。

如上所述，此沙盒强制执行两者：

1. **文件系统隔离**，允许读取和写入当前工作目录，但阻止修改其外的任何文件。
2. **网络隔离**，只允许通过连接到沙盒外运行的代理服务器的 unix 域套接字访问互联网。此代理服务器强制执行进程可连接的域限制，并处理新请求域的用户确认。如果你想要进一步提高安全性，我们还支持自定义此代理以在出站流量上强制执行任意规则。

两个组件都可配置：你可以轻松选择允许或禁止特定文件路径或域。

Claude Code 的沙盒架构通过文件系统和网络控制隔离代码执行，自动允许安全操作，阻止恶意操作，仅在需要时请求权限。

沙盒确保即使成功的提示注入也被完全隔离，无法影响整体用户安全。这样，受损的 Claude Code 无法窃取你的 SSH 密钥，或联系攻击者的服务器。

要开始使用此功能，在 Claude Code 中运行 `/sandbox` 并查看我们安全模型的 [更多技术细节](https://docs.claude.com/en/docs/claude-code/sandboxing)。

为了让其他团队更容易构建更安全的智能体，我们已 [开源](https://github.com/anthropic-experimental/sandbox-runtime) 此功能。我们相信其他人应考虑采用此技术用于他们自己的智能体，以增强其智能体的安全态势。

#### Web 版 Claude Code：在云端安全运行 Claude Code

今天，我们还发布 [Web 版 Claude Code](https://docs.claude.com/en/docs/claude-code/claude-code-on-the-web)，使用户能够在云端的隔离沙盒中运行 Claude Code。Web 版 Claude Code 在隔离沙盒中执行每个 Claude Code 会话，在其中它以安全的方式完全访问其服务器。我们设计此沙盒以确保敏感凭据（如 git 凭据或签名密钥）永远不会在沙盒内与 Claude Code 一起。这样，即使沙盒中运行的代码受损，用户也不会受到进一步伤害。

Web 版 Claude Code 使用自定义代理服务透明处理所有 git 交互。在沙盒内，git 客户端使用自定义构建的范围凭据对此服务进行身份验证。代理验证此凭据和 git 交互的内容（例如确保只推送到配置的分支），然后在将请求发送到 GitHub 之前附加正确的身份验证令牌。

Claude Code 的 Git 集成通过安全代理路由命令，验证身份验证令牌、分支名和仓库目标——允许安全的版本控制工作流，同时防止未经授权的推送。

## 开始使用

我们的新沙盒 bash 工具和 Web 版 Claude Code 为使用 Claude 进行工程工作的开发者提供了安全性和生产力的实质性改进。

要开始使用这些工具：

1. 在 Claude 中运行 `/sandbox` 并查看 [我们的文档](https://docs.claude.com/en/docs/claude-code/sandboxing) 了解如何配置此沙盒。
2. 前往 [claude.com/code](http://claude.ai/redirect/website.v1.376eb8e7-4e1c-4501-804d-66570d89b43b/code) 试用 Web 版 Claude Code。

或者，如果你正在构建自己的智能体，查看我们的 [开源沙盒代码](https://github.com/anthropic-experimental/sandbox-runtime)，并考虑将其集成到你的工作中。我们期待看到你构建的内容。

要了解有关 Web 版 Claude Code 的更多信息，查看我们的 [发布博客文章](https://www.anthropic.com/news/claude-code-on-the-web)。
